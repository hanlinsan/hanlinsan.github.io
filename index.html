Docker 使用

1. 概念

Docker是一个平台，开发者和系统管理员可以用容器来开发，部署和运行应用。在Linux中使用容器部署应用被称为容器化。容器不是新的，但使用容器来容易的部署应用是。

容器有下列特性：

  • 灵活：最复杂的应用也可以容器化
  • 轻量级：容器共享主机内核
  • 互换性：可以即时升级和更新
  • 可移植：可以本地生成，部署到云上，任何地方运行
  • 可伸缩：可以增加和自动分发容器副本
  • 可堆叠：可以在运行中将服务垂直堆叠

    Images和containers

    容器通过运行一个Image来创建。一个Image是一个可执行包，包括运行一个应用需要的一切资源：代码，运行时，库，环境变量和配置文件。
    容器是一个Image的运行时实例，Image执行时加载到内存中（包含状态，用户进程）。可以像在Linux中查看进程一样，使用docker ps来查看运行中的进程实例。

    容器和虚拟机

    Linux中的容器在本地运行，并和其它的容器共享主机内核，在分离的单独进程中运行，和其它的可执行文件一样使用内存，所以容器是轻量级的。
    虚拟机运行了一个完整的操作系统，可以访问比应用运行所需要的更多的资源。

    测试Docker环境

  1. docker --version 查看版本
  2. docker info 显示信息
  3. docker image ls
  4. docker container ls -a -q
  5. docker; docker container --help
